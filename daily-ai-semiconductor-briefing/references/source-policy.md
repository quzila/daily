# ソースポリシー

日次アップデートの収集時にこのポリシーに従うこと。

## ソース階層

### AI関連（コーディング・運用）

AI関連、特にコーディングツールの分野では、個人ブログやコミュニティの速報性が大手メディアを上回ることが多い。そのためAI関連に限り、コミュニティソースの信頼度を引き上げる。

- Tier 1（最高信頼度）：ベンダー公式発表、リリースノート、公式ドキュメント、ステータスページ。
- Tier 2（高信頼度）：
  - 定評あるテクノロジー系・ビジネス系メディア
  - **実務者ブログ・コミュニティ（Zenn、note、Reddit、Hacker News）** — 具体的な使用体験や検証結果を伴う投稿は、メディア報道と同等に扱う
  - ※ X/TwitterはAIからの検索・取得がブロックされているため、ソースとして使用不可
- Tier 3（参考情報）：未検証の噂、感想のみの投稿、ソースが曖昧なもの。

### 半導体関連

- Tier 1（最高信頼度）：ベンダー公式発表、リリースノート、有価証券報告書、IR資料。
- Tier 2（高信頼度）：編集基準を持つ定評あるテクノロジー系・ビジネス系メディア。
- Tier 3（参考情報のみ）：個人ブログ、コミュニティの解説。

## 必須の検証ルール

- 重要な主張は最低1件のTier 1またはTier 2ソースで裏付けを取ること。
- Tier 3の投稿を、製品発表・価格・性能に関する主張の唯一の根拠にしないこと。
- 決算情報にはプライマリドキュメント（IR発表、決算資料、トランスクリプト）を優先すること。
- ソース間で矛盾がある場合、両方の主張を報告し、不確実性を明記し、確信度を下げること。
- **AI関連のコミュニティソース（Tier 2）を使う際は、投稿者の実際の使用体験や具体的な検証結果が含まれていることを確認すること。**

## 鮮度ルール

- 主要ウィンドウ：対象日および直近48時間。
- 補助ウィンドウ：文脈が必要な場合は最大7日前まで。
- 古い文脈情報は「背景情報」として明確にマークすること。

## 収集パイプライン（必須）

- 必ず以下の順で実施すること：
  1. 取得（Zenn / note / Reddit / 公式）
  2. 選定（関連性と鮮度でランキング）
  3. 本文取得（タイトルやスニペットだけで要約しない）
  4. 要約（What / Why it matters / So what）
- タイトルと要約文だけから推測で本文を書くことを禁止する。
- 本文取得に失敗した記事は、確信度を下げて明示する。

## 引用フォーマット

各ソースには以下の形式を使用する：

- `[ソース名] (公開日: YYYY-MM-DD, アクセス日: YYYY-MM-DD): URL`

## 確信度の基準

- 高：一次ソースによる直接的な裏付け、または複数の高信頼ソースによる確認。
- 中：高信頼ソース1件による裏付け、または部分的な裏付けあり。
- 低：速報段階、間接的な根拠、またはソース間の矛盾が未解決。
