# レポートテンプレート

日次出力ファイルは `reports/daily-news/YYYY-MM-DD/daily-news.md` の1ファイルに統合する。
レポート完成後は `reports/daily-news/YYYY-MM-DD/index.html` を `daily-news.md` から自動生成する。

## Few-shot（良い記述例）

```markdown
### 【カテゴリA: 公式ソース（AI）】Copilot CLI が GA（GitHub Changelog, 2026-02-25）

**ひとことサマリー（1文）**: CLI上でのエージェント実行が正式化し、運用自動化の導入障壁が下がった。

**何が起きたか（What）**:
GitHub公式がGAを発表。プレビュー開始は2025-09で、MCP連携やセッション継続機能が追加された。

**なぜ重要か（Why it matters）**:
AI活用がIDE補完中心から、ターミナルを含む運用全体へ移行している。

**自分への影響（So what）**:
障害一次対応の定型コマンドをエージェント化し、調査時間短縮が見込める。

- リンク: https://github.blog/changelog/...
- 確信度: 高
```

## フォーマット固定（3/1基準）

- 記事の見出しは必ず実タイトルで書く（`### 記事1` / `### 記事2` は禁止）。
- 記事見出しは `### 【カテゴリX: ...】記事タイトル（出典名, YYYY-MM-DD）` を維持する。
- 各記事は次の4ブロックを必須とする:
  - `**ひとことサマリー（1文）**`
  - `**何が起きたか（What）**`
  - `**なぜ重要か（Why it matters）**`
  - `**自分への影響（So what）**`
- 箇条書きだけの簡略形式（`- タイトル: ...` / `- 記事全体の内容: ...`）は最終成果物では禁止。
